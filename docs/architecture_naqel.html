Architecture: No local database. WC orders are fetched live from WooCommerce REST API. Shipments are created and stored in the remote PostgreSQL via FastAPI. Naqel submission and tracking happen server-to-server (FastAPI ↔ GN Connect API).

4. GN Connect (Naqel) API Details

Connection

ProtocolREST / JSON
Base URLhttps://dev.gnteq.app
AuthBearer token (JWT, fresh per request)
Product TypesDLV (delivery) / RTN (return)

Endpoints Used

POST /api/identity/Authentication/GetToken
POST /api/gnconnect/Shipments
POST /api/gnconnect/Tracking

API Quirks & Gotchas

numberOfPiecesMust be STRING "1", not integer 1
codCurrnecyTypo in API - NOT "codCurrency"
Shipper countryISO3 (TUR, SAU, GBR)
Consignee countryISO2 (TR, SA, GB)
companyNameREQUIRED for shipper
Request bodySingle object, NOT array
Success HTTPReturns 201, not 200
Token expiryTokens expire - get fresh per request

Tracking Response Format

[ { "airwaybill": "401707257", "eventCode": "102", "eventName": "Dispatched to Destination", "event": "Dispatched to Destination", "actionDate": "2026-02-23T00:41:18+00:00", "eventCountry": "TR", "eventCity": "Istanbul", "notes": "..." }, ... ] // Returns FLAT LIST (not nested object) // Fields: eventName (not eventDescription) // actionDate (not eventDate) // eventCity+eventCountry (not eventLocation)

5. Submit Flow Detail

1. Read shipment from PostgreSQL (with items) 2. POST https://dev.gnteq.app/api/identity/Authentication/GetToken Body: { username: "***", password: "***" } Response: { token: { access_token: "eyJ...", token_type: "Bearer" } } 3. Build GN Connect payload from DB shipment: - numberOfPieces: STRING "1" (not integer!) - supplierCode: "SPL" - shipper.countryCode: ISO3 (TUR, SAU) ← converted from ISO2 - consignee.countryCode: ISO2 (JO, SA, TR) ← kept as-is - COD field typo: "codCurrnecy" (not codCurrency) 4. POST https://dev.gnteq.app/api/gnconnect/Shipments Header: Authorization: Bearer {token} Body: single object (NOT array) Success Response (HTTP 201): { "status": "Success", "airwaybill": "GNTUPD0022867326", "airwaybillId": 24877553, "shipmentLabel": "JVBERi0x..." (base64 PDF) } 5. Store in DB: - airwaybill_number = response.airwaybill - tracking_number = response.airwaybill - label_base64 = response.shipmentLabel - status = "submitted" - carrier_request_payload = full request JSON - carrier_response_payload = full response JSON

6. Tracking Flow Detail Phase 4

Single Track (POST /shipments/{id}/track)

1. Read shipment from DB 2. Get Naqel token (get_naqel_token()) 3. POST /api/gnconnect/Tracking { airwaybill, customerCode, branchCode } 4. Parse response: flat list → events[] 5. Dedup: (event_code, event_date) normalized UTC 6. Store new events in tracking_events table 7. Derive status from latest event description: - "Delivered" → delivered - "Dispatched"/"PickedUp" → in_transit - "Returned"/"Exception" → failed 8. Update shipment.status + last_tracked_at 9. Return events + current_status

Background Poller

- asyncio.Task on FastAPI startup - Runs every 3 hours - Queries: submitted/in_transit shipments with AWB - Tracks each via GN Connect - Updates status + stores events - Logs: "Polled X shipments, Y updated"

Dedup Key Normalization

Problem: DB datetime str() = "2026-02-23 00:41:18+01:00" API datetime str = "2026-02-23T00:41:18+00:00" Solution: _normalize_date_key() 1. Parse with datetime.fromisoformat() 2. Convert to UTC: .astimezone(timezone.utc) 3. Format: "%Y-%m-%dT%H:%M:%S+00:00"
