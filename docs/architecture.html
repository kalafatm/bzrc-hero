Submit to Naqel Flow (POST /shipments/{id}/submit)

1. Read shipment from PostgreSQL (with items) 2. POST https://dev.gnteq.app/api/identity/Authentication/GetToken Body: { username: "***", password: "***" } Response: { token: { access_token: "eyJ...", token_type: "Bearer" } } 3. Build GN Connect payload from DB shipment: - numberOfPieces: STRING "1" (not integer!) - supplierCode: "SPL" - shipper.countryCode: ISO3 (TUR, SAU, GBR) ← converted from ISO2 - consignee.countryCode: ISO2 (JO, SA, TR) ← kept as-is - COD field typo: "codCurrnecy" (not codCurrency) - shipper.companyName: REQUIRED (default "Bazaarica") 4. POST https://dev.gnteq.app/api/gnconnect/Shipments Header: Authorization: Bearer {token} Body: single object (NOT array) Success Response (HTTP 201): { "status": "Success", "airwaybill": "GNTUPD0022867326", "airwaybillId": 24877553, "shipmentLabel": "JVBERi0x..." (base64 PDF) } 5. Store in DB: - airwaybill_number = response.airwaybill - tracking_number = response.airwaybill - label_base64 = response.shipmentLabel - status = "submitted" - carrier_request_payload = full request JSON - carrier_response_payload = full response JSON

API Quirks & Gotchas

numberOfPiecesMust be STRING "1", not integer 1
codCurrnecyTypo in API - NOT "codCurrency"
Shipper countryISO3 (TUR, SAU, GBR) - different from consignee!
Consignee countryISO2 (TR, SA, GB) - standard
companyNameREQUIRED for shipper (even if empty company)
Request bodySingle object, NOT array
Success HTTPReturns 201, not 200
Fake-200 errorsCan return 200 with error in body
Token expiryTokens expire - get fresh per request

Gemini Translation Service

🌐
Gemini API — 3-Mode Translation
PropertyValue
Modelgemini-2.0-flash
Endpointgenerativelanguage.googleapis.com/v1beta
Temperature0.1 (deterministic)
On FailureOrder NOT imported
ModeFunctionUsed ForExample
Transliterate transliterateName() first_name, last_name, company هداية → Hedaye (NOT "Gift")
Address translateAddress() address_1, address_2, city, state شارع الملك → King Street (transliterate proper nouns)
Generic translateToEnglish() Other fields, customer notes Pure 1:1 translation
City Match matchArabicCity() Arabic city → Naqel city list عمّان → Amman (from candidate list)
Names are ALWAYS transliterated (phonetic Latin), never translated. Addresses are translated but proper nouns within are transliterated. Field routing is automatic via translateFields() based on field name. Arabic/non-English fields highlighted orange in UI. Translate button appears automatically.

City Code Fuzzy Matching Pipeline

📍
4-Step City Matching
1. Exact Match → 2. Fuzzy (Levenshtein) → 3. Gemini Arabic → 4. Operator Manual

geminiNoOperator must confirm

Currency Conversion (open.er-api.com)

💱
Free Open Exchange Rate API
PropertyValue
Sourcehttps://open.er-api.com/v6/latest/{base}
API KeyNone required (free)
Input CurrencyUSD (all WC orders)
OutputDestination currency from naqelCityCodes sheet (SAR, JOD, AED, etc.)
CacheIn-memory Map, 1-hour TTL (Next.js API route)
Update FrequencyDaily (API updates once per day)
FallbackReturns error if API unreachable
Runs entirely in the Next.js API route (/api/exchange-rates). No FastAPI backend involvement. Country currency is auto-determined from naqelCityCodes Google Sheet data. Conversion applied to customs values and COD amounts before shipment creation.
